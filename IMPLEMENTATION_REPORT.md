## 1. Изменения в модуле Students

### ✅ Добавлено поле `course` (Курс)
- **Новое поле в модели Student**: `course = models.CharField(choices=COURSE_CHOICES, max_length=20, null=True, blank=True)`
- **Возможные значения**: `1`, `2`, `3`, `4`, `graduate` (Выпускник)
- **Логика работы**:
  - ✅ Поле доступно ТОЛЬКО для категории `college` (Колледжисты)
  - ✅ При смене категории с Колледжиста на любую другую поле автоматически обнуляется
  - ✅ Поле не отображается и не валидируется для других категорий
- Добавлено в: админку, сериализаторы, список, детальную карточку, редактирование

### ✅ Исправлен процесс увольнения
- Добавлена поддержка частичной даты увольнения (только месяц и год)
- Дата увольнения теперь валидируется только при статусе `fired`
- При перетаскивании в статус Уволен можно указать точную дату или только месяц

## 2. Изменения в модуле Hr-calls

### ✅ Фильтрация по пользователю
- Добавлен фильтр `created_by` в список HR вызовов
- Поддерживается фильтрация как по `id` так и по `username` пользователя
- Пример: `GET /api/v1/hr-calls/?created_by=12` или `?created_by=ivanov`

### ✅ Детальный просмотр HR-вызова
- Полная информация по вызову: комментарии, файлы, история изменений
- Эндпоинт: `GET /api/v1/hr-calls/<id>/`

### ✅ Редактирование решенного HR-вызова
- Теперь можно редактировать любые поля даже после того как проблема помечена как решенная
- Добавлено редактирование: дата вызова, решение, комментарии, статус
- Эндпоинт: `PATCH /api/v1/hr-calls/<id>/`

### ✅ Исправлена логика вызова к HR
- Флаг `is_called_to_hr` автоматически сбрасывается только при закрытии вызова со статусом "Решен"
- Флаг остается активным если вызов просто отредактирован

### ✅ Отображение "Кот / Не кот"
- Для поля `person_type` в сериализатор добавлено `type_display`
- Значения: `Кот` и `Не кот` вместо внутренних `cat` / `nocat`
- Работает как для списка так и для детального просмотра

## 3. Изменения в модуле Analytics ✅ НОВЫЙ ФУНКЦИОНАЛ

### ✅ Разделенная аналитика
Дашборд теперь разделен на три независимые версии, все с полностью идентичным функционалом:

| URL | Описание |
|-----|----------|
| `/api/v1/dashboard/` | ✅ **Старая ручка**, все категории вместе |
| `/api/v1/dashboard/ap/` | ✅ **Только Колледжисты (АП)** |
| `/api/v1/dashboard/as-patriots/` | ✅ **Только Алабуга Старт + Патриоты** |

✅ Все три версии поддерживают:
- Фильтрацию по произвольному периоду `date_from` и `date_to`
- Распределение по уровням, статусам и категориям
- Статистику за период
- ✅ **Кликабельные метрики**

---

### ✅ Кликабельные метрики ✅ НОВОЕ
**Теперь для КАЖДОЙ цифры в аналитике возвращается готовая ссылка на список студентов которые попали в эту статистику**

Пример ответа дашборда:
```json
{
  "fired_students": 12,
  "called_hr_students": 8,
  "metrics_urls": {
    "fired": "https://api.com/api/v1/analytics/metrics/students/?metric=fired",
    "called_hr": "https://api.com/api/v1/analytics/metrics/students/?metric=called_hr",
    "new": "...",
    "active": "..."
  },
  "students_by_level": [
    {
      "level": "yellow",
      "count": 42,
      "students_url": "https://api.com/api/v1/analytics/metrics/students/?metric=level_yellow"
    }
  ]
}
```

✅ При клике на любую цифру на фронтенде достаточно сделать GET запрос по этой ссылке — получите полный список людей.

✅ Работает и для общей аналитики и для отдельных групп АП и АС+Патриоты.
✅ Работает и для метрик за выбранный период.

---

### ✅ Эндпоинт детализации метрик
| URL | Описание |
|-----|----------|
| `/api/v1/analytics/metrics/students/?metric=TYPE` | Получить список студентов по метрике |

✅ Поддерживаемые типы метрик:

| Метрика | Описание |
|---------|----------|
| `fired` | Уволенные студенты |
| `called_hr` | Вызваны к HR |
| `active` | Активные студенты |
| `new` | Новые за последний месяц |
| `status_active` | Статус Активен |
| `status_fired` | Статус Уволен |
| `level_black` | Черный уровень |
| `level_red` | Красный уровень |
| `level_yellow` | Желтый уровень |
| `level_green` | Зеленый уровень |
| `category_college` | Колледжисты |
| `category_patriot` | Патриоты |

✅ Дополнительные параметры:
- `date_from=YYYY-MM-DD` начало периода
- `date_to=YYYY-MM-DD` конец периода

Пример:
```
GET /api/v1/analytics/metrics/students/?metric=fired&date_from=2026-01-01&date_to=2026-04-20
```

✅ Ответ: массив студентов в коротком формате `StudentShortSerializer`

## 4. Изменения в Export / Import ✅ НОВЫЙ ФУНКЦИОНАЛ

### ✅ Импорт
- Добавлено поле Курс в импорт из Excel
- Автоматический маппинг значений курса
- При импорте поле корректно обновляется
- Импорт работает как upsert: если студент с таким ФИО уже существует — обновляются его данные, если нет — создается новый

### ✅ Экспорт студентов
- Добавлено поле Курс в экспорт Excel
- Корректно отображаются названия курсов
- Поле экспортируется только для Колледжистов
- Добавлена поддержка экспорта в формат CSV

### ✅ Разделенная выгрузка аналитики ✅ НОВОЕ
Теперь можно скачать аналитику отдельно по каждой группе:

| URL | Описание |
|-----|----------|
| `/api/v1/analytics/download/` | ✅ Старая ручка, полная выгрузка **всех** категорий вместе |
| `/api/v1/analytics/download/ap/` | ✅ **Выгрузка только Колледжистов (АП)** |
| `/api/v1/analytics/download/as-patriots/` | ✅ **Выгрузка только Алабуга Старт + Патриоты** |

✅ **Все три ручки поддерживают одинаковые параметры:**

| Параметр | Формат | Описание |
|----------|--------|----------|
| `date_from` | `YYYY-MM-DD` | Начало периода |
| `date_to` | `YYYY-MM-DD` | Конец периода |
| `month` | `YYYY-MM` | Выгрузка за конкретный месяц |
| `type` | `full` / `month` | Тип отчета |

✅ Примеры запросов:
```
# Выгрузка всех категорий за период
GET /api/v1/analytics/download/?date_from=2026-01-01&date_to=2026-04-20

# Выгрузка только АП за квартал
GET /api/v1/analytics/download/ap/?date_from=2026-01-01&date_to=2026-04-20

# Выгрузка только АС+Патриоты за март 2026
GET /api/v1/analytics/download/as-patriots/?month=2026-03
```

✅ Особенности:
- Формат файла, структура листов, столбцы и расчеты полностью идентичны для всех трех выгрузок
- Отличается только фильтр по категории
- Все метрики, подсчеты, агрегации рассчитываются только по своей группе
- Старая ручка продолжает работать как раньше без каких либо изменений
- Файлы имеют правильные имена с указанием группы и периода
